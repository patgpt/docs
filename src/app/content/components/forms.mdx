# Form Components

A collection of neumorphic form components including inputs, checkboxes, radio buttons, switches, and select menus.

## Text Input

Basic text input with neumorphic styling.

```html
<input type="text" class="nm-input" placeholder="Enter text..." />
```

## Interactive Examples

### Form with Validation

```jsx
function ValidatedForm() {
  const [formData, setFormData] = React.useState({
    name: "",
    email: "",
    subscription: false,
    plan: "",
  });
  const [errors, setErrors] = React.useState({});

  const validate = () => {
    const newErrors = {};
    if (!formData.name) newErrors.name = "Name is required";
    if (!formData.email) newErrors.email = "Email is required";
    if (!formData.plan) newErrors.plan = "Please select a plan";
    return newErrors;
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    const newErrors = validate();
    if (Object.keys(newErrors).length === 0) {
      // Submit form
      console.log("Form submitted:", formData);
    } else {
      setErrors(newErrors);
    }
  };

  return (
    <form onSubmit={handleSubmit} class="space-y-4">
      <div>
        <label class="block mb-2">Name</label>
        <input
          type="text"
          class={`nm-input w-full ${errors.name ? "border-red-500" : ""}`}
          value={formData.name}
          onChange={(e) => setFormData({ ...formData, name: e.target.value })}
        />
        {errors.name && <p class="text-red-500 text-sm mt-1">{errors.name}</p>}
      </div>

      <div>
        <label class="block mb-2">Email</label>
        <input
          type="email"
          class={`nm-input w-full ${errors.email ? "border-red-500" : ""}`}
          value={formData.email}
          onChange={(e) => setFormData({ ...formData, email: e.target.value })}
        />
        {errors.email && (
          <p class="text-red-500 text-sm mt-1">{errors.email}</p>
        )}
      </div>

      <div>
        <label class="nm-switch-container">
          <input
            type="checkbox"
            class="nm-switch"
            checked={formData.subscription}
            onChange={(e) =>
              setFormData({ ...formData, subscription: e.target.checked })
            }
          />
          <span class="nm-switch-label">Subscribe to newsletter</span>
        </label>
      </div>

      <div>
        <label class="block mb-2">Plan</label>
        <select
          class={`nm-select w-full ${errors.plan ? "border-red-500" : ""}`}
          value={formData.plan}
          onChange={(e) => setFormData({ ...formData, plan: e.target.value })}
        >
          <option value="">Select a plan</option>
          <option value="basic">Basic</option>
          <option value="pro">Pro</option>
          <option value="enterprise">Enterprise</option>
        </select>
        {errors.plan && <p class="text-red-500 text-sm mt-1">{errors.plan}</p>}
      </div>

      <button type="submit" class="nm-button-primary w-full">
        Submit
      </button>
    </form>
  );
}
```

          sm: "2rem",

## Accessibility

- Use proper label associations
- Include ARIA attributes where needed
- Maintain sufficient color contrast
- Support keyboard navigation
- Provide error messages
- Use fieldset and legend for groups

## Best Practices

1. **Form Layout**

   - Group related fields
   - Use consistent spacing
   - Align labels and inputs
   - Show validation state clearly

2. **Input Behavior**

   - Provide clear focus states
   - Show validation in real-time
   - Use appropriate input types
   - Include helpful placeholder text

3. **Error Handling**

   - Display clear error messages
   - Highlight problematic fields
   - Provide guidance for correction
   - Maintain form state on error

4. **Accessibility**
   - Use semantic HTML
   - Include proper ARIA labels
   - Support keyboard navigation
   - Provide error feedback
